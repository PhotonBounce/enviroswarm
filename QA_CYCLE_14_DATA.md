### Low

#### L1 ‚Äî `subscriber.py`: `start_subscriber` Still Missing `run_duration_seconds` Validation
- **Location:** `mqtt/subscriber.py`, lines 220‚Äì255
- **Description:** Cycle 13 L7 requested validation of `broker_port`, `max_queue_size`, `ingest_timeout`, `max_retries`, and `run_duration_seconds` at the top of `start_subscriber`. The current code validates the first four parameters but **omits `run_duration_seconds`**. If the function is called programmatically with a negative value (e.g., `run_duration_seconds=-1`), `time.sleep(-1)` raises `ValueError` deep in the call stack instead of failing fast at the entry point.
- **Fix Direction:** Add `if run_duration_seconds is not None and run_duration_seconds < 0: raise ValueError("run_duration_seconds must be >= 0")` at the top of `start_subscriber`.

#### L2 ‚Äî `seed_demo.py`: `main()` `args.days` Validation Inconsistent with `run_seed`
- **Location:** `seed_demo.py`, lines 610‚Äì614 and 346‚Äì347
- **Description:** `main()` validates `args.days < 0`, but `run_seed` requires `days > 0` (`days <= 0` raises `ValueError`). A user passing `--days 0` passes CLI validation but crashes inside `run_seed` with a deep-stack error instead of receiving a clear CLI boundary error.
- **Fix Direction:** Change `main()` validation to `if args.days <= 0: raise ValueError("days must be > 0")`.

#### L3 ‚Äî `seed_demo.py`: `main()` Missing `args.email` Non-Empty Validation
- **Location:** `seed_demo.py`, lines 645‚Äì649 and 352‚Äì353
- **Description:** `run_seed` validates that `email` is non-empty, but `main()` does not. Passing `--email ""` passes CLI validation and crashes inside `run_seed` with `ValueError("email must not be empty")`, producing a poor user experience.
- **Fix Direction:** Add `if not args.email: raise ValueError("email must not be empty")` in `main()`.

#### L4 ‚Äî `seed_demo.py`: `main()` Missing `args.tier` Allowed-Values Validation
- **Location:** `seed_demo.py`, lines 657‚Äì661 and 356‚Äì357
- **Description:** `run_seed` validates that `tier` is one of `{"free", "pro", "enterprise"}`, but `main()` accepts any string. Passing `--tier invalid` passes CLI validation and crashes inside `run_seed` with `ValueError("tier must be one of: free, pro, enterprise")`.
- **Fix Direction:** Add `if args.tier not in {"free", "pro", "enterprise"}: raise ValueError("tier must be one of: free, pro, enterprise")` in `main()`.
